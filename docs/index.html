Installation einer Angular-Webapp in Visual Studio Code / Visual Studio

1. Visual Studio Code / Visual Studio und Node.js installieren

Visual Studio Code

  • Besuche die offizielle Visual Studio Code Webseite und lade den Installer für dein Betriebssystem herunter.
  • Installiere Visual Studio Code, indem du dem Installationsassistenten folgst.

Visual Studio

  • Besuche die offizielle Visual Studio Webseite und lade den Installer für dein Betriebssystem herunter.
  • Wähle bei der Installation die benötigten Workloads (z.B. "ASP.NET and web development").
  • Installiere Visual Studio, indem du dem Installationsassistenten folgst.

Node.js und npm

  • Besuche die offizielle Node.js Webseite und lade die LTS-Version für dein Betriebssystem herunter.
  • Installiere Node.js, indem du dem Installationsassistenten folgst. npm wird automatisch mitinstalliert.

2. Im IDE Terminal folgende Befehle ausführen

Visual Studio Code

Windows, macOS und Linux:

  1. Öffne das AngularBootcampApp Projekt über den Ordner mit VSCode
  2. Terminal des Clients in Visual Studio Code öffnen
    • Öffne Visual Studio Code.
    • Gehe zu View > Terminal oder drücke Ctrl + ` (Windows/Linux) oder Cmd + ` (macOS).
  3. Node.js Version prüfen 
    node -v
  4. npm Version prüfen 
    npm -v
  5. Angular CLI installieren 
    npm install -g @angular/cli
  6. Anwendung starten 
    ng serve
    Das erste Mal starten dauert immer etwas.

    Falls folgender Fehler auftritt: 
    ng:Die Datei kann nicht geladen werden, da die Ausführung von Skripts auf diesem System deaktiviert ist...
    Im Terminal:
    Dropdown der Powershell öffnen und Eingabeaufforderung/Commandprompt auswählen.

Visual Studio

Windows und Linux:

  1. Node.js Tools für Visual Studio installieren
    • Stelle sicher, dass die "Node.js development"-Workload bei der Installation von Visual Studio ausgewählt ist. Andernfalls kannst du diese über den Visual Studio Installer nachträglich hinzufügen.
  2. Terminal des Clients im Projekt öffnen
    • Führe die AngularBootcampApp.sln mit Visual Studio aus.
    • Gehe zu View > Terminal oder drücke Ctrl + ` (Windows/Linux).
  3. Node.js Version prüfen 
    node -v
  4. npm Version prüfen 
    npm -v
  5. Angular CLI installieren 
    npm install -g @angular/cli
  6. Anwendung starten
    Es gibt mehrere Möglichkeiten den Code zu starten:
    ng serve
    Falls folgender Fehler auftritt: 
    ng:Die Datei kann nicht geladen werden, da die Ausführung von Skripts auf diesem System deaktiviert ist...
    Im Terminal:
    Dropdown der Powershell öffnen und Eingabeaufforderung/Commandprompt auswählen.

    oder
    beim Starten Button das DropDownMenü öffnen und konfigurieren auswählen
    Bei Startprojekt: Einzelnes Startprojekt auswählen und AngularApp1.Server wählen
    Übernehmen + Ok Danach im DropDownMenü https_app auswählen und das Projekt starten. Das erste Mal starten dauert immer etwas.

Schritt 3: Erstellen einer neuen Component

  1. Öffnen Sie das Terminal (insofern Sie es geschlossen haben).

    Es gibt zwei Schreibweisen für viele Kommandos: 
    ng generate component name
    ng g c name

    Der Name unserer Component lautet todo. 
    ng g c todo

    Diese ist in src/app/todo auffindbar.
    Jede Component besteht aus
      component.css (Styling)
      component.html (Struktur)
      component.spect.ts (Tests)
      component.ts (Funktionalität und Logik)
    Diese Components finden sich auch in der app.component.
    In der app.component.html wird bestimmt welche Komponenten beim Starten der Component sichtbar sind.

    Fügen Sie folgenden Code ein: 
    import { Todo } from '../models/todo.module';
import { TodoService } from '../services/todo.service';
    Falls die Imports nicht funktionieren sollten könnten diese auch von VSCode so erwartet werden: 
    import { Todo } from '../../models/todo.module';
import { TodoService } from '../../services/todo.service';
    todos: Todo[] = [];
newTodoTitle: string = '';

constructor(private todoService: TodoService) {
    this.todos = this.todoService.getTodos();
}

addTodo(): void {
    if (this.newTodoTitle.trim()) {
    this.todoService.addTodo(this.newTodoTitle.trim());
    this.newTodoTitle = '';
    this.todos = this.todoService.getTodos();
    }
}

deleteTodo(id: number): void {
    this.todoService.deleteTodo(id);
    this.todos = this.todoService.getTodos(); // Aktualisieren der To-Do-Liste
}

Schritt 4: Erstellen eines neuen Services

  1. Erstellen Sie einen neuen Ordner 
    cd src/app
    mkdir services
  2. Erstellen Sie einen Service im neuen Ordner 
    ng generate service services/name
    ng g s services/name
    Unser Service heißt ebenfalls todo. 
    ng g s services/todo
  3. Fügen Sie folgenden Code im todo.service.ts ein:
    Visual Studio: 
    import { Todo } from '../models/todo.module';
    Visual Studio Code: 
    import { Todo } from '../../models/todo.module';
    private todos: Todo[] = [];
private idCounter = 0;
                    
constructor() { }
    getTodos(): Todo[] {
return this.todos;
}
    addTodo(title: string): void {
const newTodo: Todo = { id: this.idCounter++, title };
this.todos.push(newTodo);
}

Schritt 6: app.module.ts

  1. Die app.module.ts ist der Kern der Angular-App. Diese wird allerdings nur erstellt wenn das Projekt mit Visual Studio erstellt wurde.
    In dieser Datei werden alle Importe vollzogen. Ist eine Componente nicht dort importiert kann diese auch nicht aufgerufen und genutzt werden.
    Folgende Imports sollten nun auch darin enthalten sein: 
    import { TodoComponent } from 'dein jeweiliger Path';
import { TodoService } from 'dein jeweiliger Path';
  2. Durch den Import können Sie nun TodoComponent und TodoService aufrufen.

    TodoComponent wird bei declarations hinzugefügt.
    TodoService wird bei den providers hinzugefügt.

Schritt 13: Löschen einer Todo

    Fügen Sie folgenden Code in der todo.service.ts hinzu: 
    deleteTodo(id: number): void {
const index = this.todos.findIndex(todo => todo.id === id);
if (index !== -1) {
this.todos.splice(index, 1);
    }
}
    und erweitern Sie die todo.component.html an der richtigen Stelle um folgenden Code: 
    <button class="delete" (click)="deleteTodo(todo.id)">
    <i class="material-icons">delete</i>
</button>
    Installieren Sie folgendes über das Client-Terminal: 
    ng add @angular/material

Flexibilität

    Die CSS lässt sich je nach Bildschirmgröße verändern. Um diese richtig anzeigen zu können wird @media(Breakpoint wird hinzugefügt) verwendet.
    Diesen Code können Sie hinzufügen um das Styling für Bildschirmgrößen bis zu einer maximalen Breite von 1024px; 
    /* Tablet styles */
@media (max-width: 1024px) {
    h1 {
        text-align: center;
        font-size: 24px;
    }

    input {
        width: calc(100% - 100px);
        font-size: 14px;
    }

    button {
        height: 40px;
        width: 40px;
    }

    li {
        width: calc(100% - 100px);
        font-size: 14px;
    }
}
